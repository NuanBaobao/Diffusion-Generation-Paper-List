## Qwen-Image-VAE-2.0 Technical Report
[https://arxiv.org/abs/2605.13565](https://arxiv.org/abs/2605.13565)


<p align="center">
<img src="https://cdn.nlark.com/yuque/0/2026/png/26304400/1779182172323-f7a71c88-e534-44d0-b70a-4a6e0e89773d.png" width="50%">
</p>


**Qwen-Image-VAE-2.0**，目标是在 **f16 / f32 高压缩率** 下同时保持重建质量、文字可读性和下游 diffusion 友好性。它通过 **大 latent channel、Global Skip Connection、大规模文本/合成数据、DINOv2 中间语义对齐** 来缓解高压缩 VAE 的信息丢失和 latent 难建模问题


## Spectral Progressive Diffusion for Efficient Image and Video Generation
[https://arxiv.org/abs/2605.18736](https://arxiv.org/abs/2605.18736)


<p align="center">
<img src="https://cdn.nlark.com/yuque/0/2026/png/26304400/1779283915501-04a1177e-f9ce-4e35-a79f-4768c804a40b.png" width="50%">
</p>


**Spectral Progressive Diffusion** 提出了一种基于频率域的 progressive-resolution diffusion 加速框架。它利用扩散模型“低频先生成、高频后生成”的 spectral autoregression 特性，在 denoising 早期只用低分辨率计算，等高频信号开始出现时再通过 **spectral noise expansion** 升分辨率，并用 **timestep alignment** 保证状态仍处于正确的 flow matching 轨迹上。作者进一步根据 power spectrum 推导出 **δ-optimal resolution schedule**，用一个误差容忍参数 $\delta$ 自动决定升分辨率时间。

## GAE:Geometric Autoencoder for Diffusion Models
[https://arxiv.org/abs/2603.10365](https://arxiv.org/abs/2603.10365)


<p align="center">
<img src="https://cdn.nlark.com/yuque/0/2026/png/26304400/1776508778139-21382302-6cef-4618-8efc-4641dd991fd7.png" width="50%">
</p>


GAE 是一个为 diffusion model 专门设计的 autoencoder。它用 DINOv2 语义特征训练一个低维 semantic teacher，然后直接**监督 autoencoder 的 latent bottleneck**；再用 Latent Normalization 和 Dynamic Noise Sampling 让 latent 分布稳定、decoder 对噪声鲁棒。最终得到的 latent 低维、语义强、重建稳定，因此能显著加速和提升后续 diffusion 训练

## MacTok: Robust Continuous Tokenization for Image Generation
[https://arxiv.org/abs/2603.29634](https://arxiv.org/abs/2603.29634)


<p align="center">
<img src="https://cdn.nlark.com/yuque/0/2026/png/26304400/1778113740175-6b49acff-2cf1-450d-b686-82afb772b545.png" width="50%">

## VTP:Towards Scalable Pre-training of Visual Tokenizers for Generation
[https://arxiv.org/abs/2512.13687](https://arxiv.org/abs/2512.13687)

视觉 tokenizer / VAE 的预训练是否能像大模型预训练一样 scale，并持续提升下游生成质量？


<p align="center">
<img src="https://cdn.nlark.com/yuque/0/2026/png/26304400/1779281484467-9cd39508-d430-4569-a004-39e3c6b64fe6.png" width="50%">
</p>


**VTP** 这篇文章提出了一个很清晰的观点：传统视觉 tokenizer 只靠 reconstruction pre-training 无法有效 scale 到生成任务，因为它会越来越偏向低级像素细节，而不是 generation 所需的高级语义结构。作者提出 **Visual Tokenizer Pre-training**，把 tokenizer 训练成一个 perception-oriented representation learner，同时优化 **reconstruction、self-supervised learning 和 CLIP-style contrastive learning**。实验表明，传统 AE 随预训练 FLOPs 增加只会提升 rFID，却让 gFID 变差；而 VTP 随 compute、model size 和 data scale 增加持续提升下游 generation。最终 VTP 达到 **0.36 rFID、78.2% zero-shot、85.7% linear probing accuracy**，并让标准 LightningDiT 在 ImageNet 256×256 上达到 **1.11 gFID**。

# 2510
## HyCa:Let Features Decide Their Own Solvers: Hybrid Feature Caching for Diffusion Transformers (ICLR 2026)
[https://arxiv.org/abs/2510.04188](https://arxiv.org/abs/2510.04188)


<p align="center">
<img src="https://cdn.nlark.com/yuque/0/2026/png/26304400/1776357713099-ce0c4704-f524-45aa-a6c1-8e843f3dcc31.png" width="50%">
</p>


主要是用来在扩散 Transformer 采样时做**特征缓存（feature caching）**。它的关键思想不是“所有特征都用同一种预测/缓存策略”，而是让**不同特征维度根据自己的动态特性，选择最合适的数值求解器**来预测下一步特征，从而在基本不掉质的情况下显著加速采样

This paper proposes a training-free diffusion transformer acceleration framework, HyCa, which models hidden feature evolution as a mixture of ODEs and performs dimension-wise hybrid solver assignment for feature caching, yielding strong speed-quality trade-offs across image, video, editing, and distilled settings

## PPFlow：Pyramidal Patchification Flow for Visual Generation
[https://arxiv.org/pdf/2506.23543](https://arxiv.org/pdf/2506.23543)


这篇论文的动机是**提升扩散变换器（DiT）的推理效率**，特别是减少去噪过程中输入到DiT块的token数量，从而降低计算成本。

其核心创新点是一种名为**金字塔式分块流（Pyramidal Patchification Flow, PPFlow）**的方法，具体包括：

1. **动态分块策略**：根据噪声水平（时间步）动态调整Patchify操作中的块大小。高噪声时用**大块**（如4x4），低噪声时用**小块**（如2x2），以减少高噪声阶段（计算量主要集中于此）的token数量。
2. **全分辨率操作**：与需要在不同分辨率金字塔表征间转换的Pyramidal Flow等方法不同，PPFlow始终在**全分辨率潜在表征**上进行操作，避免了复杂的“再噪声（renoisng）”技巧和“跳跃点”问题。
3. **共享与独有参数**：不同块大小级别**共享DiT块的全部参数**，但各有其**独立的Patchify和Unpatchify线性投影层参数**。
4. **高效训练方式**：支持**从头训练**和**基于预训练DiT微调**两种模式。微调时通过巧妙的参数初始化，能用极少的额外训练成本（<10% FLOPs）获得高性能模型。

**最终效果**：在保证甚至略微提升图像生成质量（FID, IS）的同时，实现了**1.6倍至2.0倍的推理加速**。

## REPA:Representation Alignment for Generation: Training Diffusion Transformers Is Easier Than You Think (ICLR2025 Oral)
[https://arxiv.org/abs/2410.06940](https://arxiv.org/abs/2410.06940)

训练 DiT / SiT 这类 diffusion transformer 的困难，不只是去噪建模本身，而是模型需要在训练过程中同时学到高质量视觉表征；如果用外部 pretrained visual encoder 的表征来辅助对齐，生成模型会更快收敛、质量更好


<p align="center">
<img src="https://cdn.nlark.com/yuque/0/2026/png/26304400/1779409514949-cea915f0-b16b-49c6-b372-ba95adf2b5cc.png" width="50%">
</p>


**REPA** 用 DINOv2、SigLIP、CLIP、MAE 等 pretrained visual encoder 的 clean image representation 来监督 diffusion transformer 的中间 hidden states，使模型从 noisy latent 中学到更强、更稳定的视觉表征。它的核心判断是：**DiT / SiT 训练慢的重要原因是内部 representation learning 不充分，而外部强视觉表征可以显著降低这个学习难度。**
